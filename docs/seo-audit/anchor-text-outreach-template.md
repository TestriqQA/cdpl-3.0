# Anchor-Text Outreach Template (BLG-203)

> **Owner:** Content / Marketing / Founder outreach
> **Backlog:** BLG-203 (Phase 10d follow-up)
> **Last updated:** 2026-05-28
> **Goal:** when CDPL pays for, applies for, or earns an external mention, ask the publisher to use **descriptive keyword anchor text**, not the bland brand-only anchors that currently dominate the link profile.

---

## Why this matters

Phase 10d ([`10d-blg-189-gsc-links.md`](10d-blg-189-gsc-links.md)) pulled the GSC Links report. The top 5 external anchors:

| # | Anchor text | Read |
| --- | --- | --- |
| 1 | `https cinutedigital com` | URL-shaped |
| 2 | `www cinutedigital com` | URL-shaped |
| 3 | `cinutedigital com` | URL-shaped |
| 4 | `cinute digital` | Brand |
| 5 | `certification preparation` | **Only real keyword anchor** |

**4 of top 5 anchors are URL-shaped or pure brand.** This is the signature of a link profile dominated by:

- Scraper/automated sites that paste the URL as the anchor (the websitedesing.com.tr footprint — see `disavow.txt`)
- Social shares (LinkedIn) where the platform auto-renders the URL
- Generic directory listings that use the company name as the link text

The signal Google takes from this profile: **CDPL is associated with the brand name "cinute digital"**, but NOT with the commercial keywords its courses actually target (`software testing course`, `data science training`, `digital marketing course`, etc.). Branded queries are already won (position 1-3); commercial queries are stuck at the position-16 average from Phase 10b.

**Keyword-anchor inbound links feed Google's understanding of WHAT the page ranks for**, beyond what on-page content alone can signal. Every keyword anchor CDPL earns moves the needle on commercial-query rankings.

You can't control what every site writes. But for every outreach where CDPL gets to write the listing or request the anchor (directory submissions, award entries, podcast/interview appearances, review platforms, partnership announcements, guest posts), **pre-write the desired anchor text**. Publishers paste what you give them ~80% of the time.

## The anchor-text priority list

Use these in the order shown — top of the list = highest commercial value + lowest competition. When you have ONE link to request, ask for #1. When you have multiple links from the same publisher, distribute across the list.

### Tier 3 — safe brand anchors (fall-back only)

Use only when the publisher refuses descriptive anchors (some news outlets, academic publications, GitHub READMEs). These are the same anchors GSC already overflows with — adding more doesn't hurt but doesn't move the needle either.

- `Cinute Digital Pvt. Ltd.`
- `Cinute Digital`
- `CDPL`

### NEVER use

- `click here` / `read more` / `learn more` (zero SEO value, also accessibility-hostile)
- Empty anchor (just an image link without `alt`)
- The bare URL (`https://www.cinutedigital.com/`) — Google already classifies these as URL-shaped via the auto-detection; doesn't add keyword signal
- "5,000+ placed", "100% placement", "15+ years" or any other non-defensible-stat claim from `_decisions.md` — these are forbidden in all new copy, not just JSON-LD

## Per-channel templates

### A. Directory submission (Sulekha / Justdial / Tracxn / Crunchbase / Clutch / G2 / Shiksha / CollegeDunia)

These are BLG-194 / BLG-184 / BLG-197 / BLG-198 / BLG-199 / BLG-130 targets. Most directories let you write the listing description yourself.

**Description template** (use Tier 1 anchors as natural links inside the description):

```
Cinute Digital Pvt. Ltd. (CDPL) is an ISTQB Training Partner based in
Mira Road, Mumbai, offering industry-led courses including
[Software Testing Course in Mumbai by Cinute Digital], [Data Science
Course in Mumbai by Cinute Digital], [Digital Marketing Course in
Mumbai by Cinute Digital], and [Power BI Training Mumbai — Cinute Digital].
Founded in 2020 by Sandeep Maske, CDPL is rated 4.9/5 across 425 verified
reviews. The institute is AAA and ACTD accredited and provides placement
assistance, mock interviews, and referrals to hiring partners.

Visit https://www.cinutedigital.com to enroll.
```

Bracketed `[text]` = the anchor text. Replace with the directory's link-input field as appropriate.

### B. Award submission / EdTech recognition (BLG-191)

Most award entries have a 1-paragraph "about the institution" + a separate "submission link" field.

**About field:**

> Cinute Digital Pvt. Ltd. (CDPL) is a Mira Road, Mumbai-based EdTech institute providing hands-on, mentor-led courses in Software Testing, Data Science, Machine Learning, Business Intelligence, AI, and Digital Marketing. ISTQB Training Partner. AAA + ACTD accredited. 4.9/5 across 425 verified student reviews.

**Link-back field:**

If single link: request anchor **"Software Testing Course in Mumbai by Cinute Digital"** → `/courses/software-testing-course`.
If the award has a "category" they list, request the anchor that matches their category (data-science award → use Tier 1 #2; digital-marketing award → Tier 1 #3, etc.).

### C. Podcast / interview / speaking-circuit appearance (BLG-192)

Most podcast / event sites render the speaker's name as a link to their company. Request:

- **Speaker name link** → `https://www.linkedin.com/in/sandeepmaske/` (founder LinkedIn — already in `Org.founder.sameAs`)
- **Company name link** (if separate) → use a Tier 1 anchor descriptive of the topic discussed. E.g., a software-testing-themed podcast: anchor = "Software Testing Course in Mumbai by Cinute Digital".

### D. Review platform listing (Trustpilot / Glassdoor / Google Reviews / Justdial reviews — BLG-193)

These have a "Website" or "Company URL" field rather than free anchor-text. Use the **canonical** URL only:

- `https://www.cinutedigital.com/` (no trailing path)
- Never `cinutedigital.com` (no protocol — wastes the link)
- Never `www.cinutedigital.com` (no protocol — same problem)
- Never the non-www variant — domain canonical is `www.`

### E. Partnership announcement (MoU signing / FDP / college collaboration — bhavans.ac.in pattern)

When CDPL signs an MoU with an institution and they publish a press release / announcement, request:

- First mention: **Cinute Digital Pvt. Ltd. (CDPL)** linked to `https://www.cinutedigital.com/about-us`
- Subsequent mention of the program: the Tier 1 anchor that matches the program — e.g. for an FDP on Power BI: **"Power BI Training Mumbai — Cinute Digital"** → `/courses/bi-courses/power-bi-course`

This pattern already partially works — `bhavans.ac.in` has 1 backlink to CDPL from an MoU announcement (Phase 10d data). Future MoU partners should be sent the suggested anchors at the time of the press release.

### F. Guest post on a tech blog / Medium article (BLG-104 adjacent)

If a CDPL employee/founder writes a guest post on a tech publication:

- Author bio (end of article): **Sandeep Maske is the founder of [Cinute Digital](https://www.cinutedigital.com/about-us), an EdTech institute in Mira Road, Mumbai.**
- In-article reference (if natural): **"At [Cinute Digital](https://www.cinutedigital.com/), we run a [Software Testing Course in Mumbai](/courses/software-testing-course) ..."** — relative path for internal CDPL articles, absolute for guest posts.

## Implementation checklist for outreach owner

Before pasting an anchor into a directory / award form / partnership announcement, run through this 3-step check:

1. **Look up Tier 1.** Pick the anchor that matches the closest commercial keyword for the receiving page.
2. **Confirm the destination URL is the canonical** form (with `www.`, with `https://`, no trailing slash for category pages, trailing slash for the homepage only). Cross-check against `next.config.ts` and `_decisions.md`.
3. **Confirm the description does NOT contain forbidden non-defensible-stat claims** ("5,000+ placed", "100% placement", "15+ years" — see `_decisions.md`).

Only stat that's safe in any new copy: **4.9 / 5 (425 verified reviews)**.
